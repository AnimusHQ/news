# Script Skeleton: What happens after `git push`?

## Hook

You typed `git push`. A few minutes later, your code may be running in production. But between those two moments, a whole engineering system wakes up.

## Promise

In this episode, we follow the path from local commit to remote repository, CI checks, build artifacts, deployment, rollback, and observability.

## Outline

1. Local commits and remote repository update.
2. Repository event triggers automation.
3. CI validates the change.
4. Build artifacts or container images may be produced.
5. Deployment strategy moves the change toward production.
6. Observability tells the team whether reality matches expectations.
7. Rollback exists because production is never theoretical.
