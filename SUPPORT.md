# Support

AnimusHQ repositories are open-source projects with best-effort maintenance unless a repository states a different support model.

## Questions and discussions

Use GitHub issues only for actionable work:

- reproducible bugs;
- documentation defects;
- build or test failures;
- scoped feature requests;
- security-hardening tasks that do not disclose vulnerabilities.

For architecture review, collaboration, or consulting inquiries, use email:

```text
rewanderer@proton.me
```

## Security issues

Do not report vulnerabilities in public issues.

Follow `SECURITY.md` and report privately by email.

## Not supported by default

Unless a repository explicitly says otherwise, AnimusHQ does not provide:

- production deployment support;
- incident response;
- managed hosting;
- compatibility guarantees;
- long-term maintenance guarantees;
- security certification;
- SLA-backed response times.

## What makes a good issue

A good issue includes:

- repository and version or commit;
- operating system and environment;
- expected behavior;
- actual behavior;
- reproduction steps;
- logs or traces without secrets;
- minimal configuration needed to reproduce.
