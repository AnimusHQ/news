## Task pack

Task ID: CFG-001

Title: Content-agnostic provider configuration cleanup

Goal: ensure provider configuration does not hardcode a video topic in `.env`, wrappers,
scripts, tests, or production code, and that provider live calls are explicitly
guarded.

Scope:

- `.env.example`;
- `cmd/animus-news`/pilot CLI behavior inspection only;
- `internal/shortform/pilot`;
- Claude API review provider live-call guard;
- provider wrapper examples under `scripts/providers`;
- focused provider/runbook docs;
- CFG ledger/report.

Non-goals:

- no public publishing;
- no request-file feature because CLI flags already provide the required runtime
  content input;
- no generated media committed;
- no live provider call without operator prompt and live configuration;
- no unrelated connector or workflow redesign.

Parallelization plan:

- Lane A inspected CLI/runtime content inputs.
- Lane B inspected provider wrappers and `.env` policy.
- Lane C inspected the regression-check location.
- Integration owner applied and verified the final patch set.

## Implementation notes

Implemented:

- `pilot generate-real` remains CLI-flag based. No request-file feature was added
  because existing flags already cover episode id, prompt, language, duration,
  platforms, provider choices, Claude review mode, and output directory.
- Removed open-source / ecosystem themed fallback script and shot prompt text
  from `internal/shortform/pilot/helpers.go`.
- Removed the defensive `youtube` platform default from pilot YAML output.
- Added regression tests in `internal/shortform/pilot` for content/config
  separation.
- Added `ANIMUS_ALLOW_LIVE_PROVIDER_CALLS=1` guard to the Claude API env-built
  client and to both example wrappers.
- Added wrapper output-root containment checks before writing files.
- Updated `.env.example` to contain provider/operation placeholders only.
- Updated runbooks and provider docs to use runtime shell variables and
  placeholders instead of a fixed episode id or topic.

Not run:

- The real MVP smoke/full live run is intentionally out of scope for CFG-001 and
  must run separately after this configuration cleanup is committed.

Planned:

- A future request-file path can be added only if richer content inputs
  (audience, style, CTA, per-shot prompts) are required beyond the current CLI
  flags.
