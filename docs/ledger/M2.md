Assumptions:

- The user prompt is the M2 task pack and supersedes M1's mock-only wording where
  it explicitly permits local adapters.
- Real adapters remain disabled by default; default `make verify` must stay
  offline, deterministic, and zero-spend.
- Local FFmpeg tests may skip when the `ffmpeg` binary is unavailable.
- The faster-whisper sidecar boundary can be contract-first and disabled by
  default; it must not download models during verification.
- Upload-Post in M2 is a request/manifest/dry-run contract only; live mode must
  return a clear error.

Risks:

- FFmpeg output determinism can vary by binary build and codec availability; M2
  should assert contract properties rather than byte-identical media output.
- `exec.CommandContext` timeout tests must avoid flaky long sleeps.
- Path validation must reject traversal while still allowing test temp
  directories and controlled run output directories.
- Temporal JSON-history replay may remain gated if no dev-server history fixture
  can be recorded safely offline.
