# M1 Work Ledger — OpenShorts → AnimusHQ/news Integration

> Running ledger for Milestone M1. Append-only by intent. Every non-trivial
> decision links to an ADR under `docs/adr/`. State must be reconstructable by
> another engineer from this file + the ADRs alone (takeover test).

Mission (bounded): integrate the **execution capabilities** referenced from
OpenShorts (subtitles, FFmpeg render/composite, 9:16 normalization, Upload-Post
publishing pattern) into the News control plane as **typed, validated, gated
contracts** that run **end-to-end on mock providers**, with every quality/release
gate enforced in code, every artifact typed/validated/hashed/versioned, and the
whole thing reproducible by a single verify command. M1 proves architecture,
contracts, and gates — **not** real provider execution (that is M2/M3).

### Delta list (this document ↔ repo reality)

D-01. **CLI path.** Driver doc says `go run ./cmd/news demo …`; the real binary is
`cmd/animus-news`. **Resolution:** add the `demo` subcommand to `cmd/animus-news`;
document the exact command in the report. No `cmd/news` created (avoid surface).

D-02. **Artifact status model.** Repo has 4 statuses; M1 envelope requires
`draft|in_review|approved|rejected|superseded|locked` (adds `in_review`, `locked`).
**Resolution:** extend `ArtifactStatus` + `validArtifactStatus` with the two new
states (backward-compatible; existing tests only assert draft/approved + reject
unknown). See ADR-0002.

D-03. **Canonical content hashing.** No deterministic canonical-JSON hash that
excludes the `content_hash` field exists. M1 §7 requires one. **Resolution:** new
`internal/shortform/contenthash` computing sha256 over canonicalized (sorted-key)
JSON with the hash field removed. See ADR-0002.

D-04. **JSON Schema.** Repo validates by hand-written Go; no JSON-Schema library is
vendored, and M1 requires "Go types + JSON Schema + validators" while also
requiring offline reproducibility (no network egress to pass tests). **Resolution:**
ship real JSON Schema documents as committed contracts AND a small embedded
schema interpreter (no new dependency) that validates instances against them, so
the schema is load-bearing for the accept/reject tests. See ADR-0003.

D-05. **New short-form artifacts are additive, not part of the required bundle.**
The existing `episode-0001` bundle does not contain the 8 new artifacts; adding
them to `RequiredEpisodeFiles` would break existing validation. **Resolution:**
implement them in a dedicated `internal/shortform` package and a separate run
directory; register single-file recognition only where safe. See ADR-0001.

D-06. **Disclosure field naming.** Existing publishing uses
`synthetic_disclosure_required`; M1 §4.8/§7 uses `ai_disclosure_required`.
**Resolution:** new artifacts use `ai_disclosure_required` (matches M1 + OpenShorts
spec). Legacy field left untouched. Recorded as terminology delta.

D-07. **Workflow shape.** The existing `EpisodeLifecycleWorkflow` is long-form and
does not cover the §9 short-form activity list. **Resolution:** add a dedicated
`ShortFormWorkflow` + activities; leave the existing workflow intact. See ADR-0004.

D-08. **Demo vs. live Temporal.** A live Temporal server is not available offline,
but the e2e demo (§Phase 5) must be runnable by a single command. **Resolution:**
the demo runs an in-process pipeline runner that shares the exact activity/gate
functions with the Temporal workflow; the durable workflow itself is proven by the
Temporal `testsuite` env + a replay test (no server needed). See ADR-0004.

## Phase 1 — Artifact contracts (types + schema + validation + hashing)

Implemented in `internal/shortform`:

- **Typed structs** (`artifacts.go`): all 8 short-form artifacts embed the common
  `Envelope` and implement `Artifact` (`Kind()`, `EnvelopeRef()`). Shared value
  objects: `ProviderRef`, `RenderTarget`, `MediaOutput`, `RendererRef`,
  `RenderOutput`, `RiskAcceptance`, `SubtitleChecks`, `CandidateComponent`.
- **Content hashing** (`contenthash/`): `Compute`/`Verify`/`Canonicalize` — sha256
  over canonical (sorted-key) JSON with the `content_hash` field excluded. `Stamp`
  writes the hash onto an artifact idempotently.
- **JSON Schema** (`schemas/*.schema.json`, embedded via `go:embed`) + a
  dependency-free Draft-2020-12-subset interpreter (`schema/`) that fails closed on
  unsupported keywords.
- **Validation** (`validate.go`): `Validate(Artifact)` = schema + envelope semantics
  (RFC3339 created_at, created_by form, sha256 prefix) + light kind semantics
  (unique scene_ids; aspect match; production_candidate locked+immutable).
  `ValidateFile` adds full content-hash integrity from bytes. `ValidateBytes`,
  `KindFromFilename`, `SchemaFor`, `KnownKinds` exposed for CLI/demo.
- **Status model** (delta D-02): added `in_review`, `locked` to
  `internal/artifacts/types.go` + `validArtifactStatus`; added
  `ArtifactStatus.IsTerminalImmutable()`.

CLI: `animus-news validate-shortform <file>` validates one short-form artifact.

## Phase 4 — Temporal workflow + activities

- **Activities** (`internal/shortform/activities`): the §9 activity set as methods on
  `*Activities` (provider set injectable for defects). Includes the deterministic
  prompt pack, import/validate/approve storyboard, generate/approve visual shots,
  voiceover, subtitles (+validate), render, production QA, candidate assembly, release
  approval, publish manifest (+validate), and the guarded dry-run. The M2/M3 paths
  (`GenerateSeedanceShots`, `UploadPostSchedulePublish`) return a
  "deferred to M2/M3" error — proving they never run in M1.
- **Workflow** (`internal/workflows/shortform.go`): `ShortFormWorkflow` orchestrates the
  full pipeline via `workflow.ExecuteActivity`, running the pure gates inline at each
  transition. Human approvals are Temporal **signals**: `StoryboardImageApproval`,
  `ReleaseApproval`. State is exposed via the `GetShortFormState` query. Registered in
  `internal/worker/worker.go` alongside the existing long-form workflow.
- **Serialization boundary**: production-candidate assembly passes concrete
  `shortform.ComponentRef` descriptors (not interface values) so nothing interface-typed
  crosses Temporal's data converter.
